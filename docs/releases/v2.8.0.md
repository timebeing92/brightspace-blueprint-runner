# Blueprint Wizard v2.8.0

Blueprint Wizard turns a Brightspace/D2L course export into a comprehensive,
review-ready course blueprint—including Word and Markdown documents, activity
and rubric files, QA reports, and a source-traceable run receipt. It runs on
your own computer and uses no AI.

Version 2.8.0 makes the Wizard easier to keep current for colleagues who use
release ZIPs. It introduces a stable managed launcher while keeping the
original portable one-folder release available.

## Which download should I choose?

**Most users:** download `blueprint-wizard-managed-v2.8.0.zip`.

1. Unzip the download.
2. On macOS, open `Blueprint Wizard.command`; on Windows, open
   `Blueprint Wizard.bat`.
3. Follow the prompts and drag in a Brightspace export when asked.

The first run checks the machine and, with permission, creates a private Python
environment for the included extraction tools. Course exports and generated
blueprints remain on the user's computer.

**Existing portable users:** `blueprint-wizard-v2.8.0.zip` remains fully
supported. Portable folders show update notices but do not replace themselves.
This same portable ZIP is also the verified version payload used internally by
managed updates.

## What is new in v2.8.0?

- The managed package keeps one durable top-level launcher while complete
  Wizard versions are installed side by side.
- Settings, logs, update state, receipts, and generated outputs live under
  `user-data/`, outside replaceable version directories.
- A future update is downloaded only after confirmation and is verified against
  GitHub's asset digest, the published SHA-256 sidecar, repository and commit
  identities, archive layout, critical runtime files, and all six portable
  schema hashes before activation.
- Activation uses an atomic pointer update and one restart. The prior complete
  version remains available for rollback, and old-version removal is explicit
  and protects current or unproven rollback state.
- The paired Bundle v1.3.0 adds flexible activity and structure contracts,
  exact run identity, direct and package-HTML syllabus-link discovery, and a
  regression gate for headings wrapped in inline elements such as `strong` or
  `span`.
- Recoverable component failures continue to preserve usable deliverables and
  report clearly what needs review instead of suppressing the entire result.
