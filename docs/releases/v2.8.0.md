# Blueprint Wizard v2.8.0

Blueprint Wizard v2.8.0 introduces the stable managed launcher while retaining
the portable one-folder release used by earlier versions. Both distributions
contain one immutable Runner/Bundle pair: Runner v2.8.0 and Blueprint Bundle
v1.3.0.

## Managed installation

- New users can keep one durable top-level launcher while complete versions
  are installed side by side under `versions/`.
- Settings, logs, update state, receipts, and generated outputs live under
  `user-data/`, outside replaceable version directories.
- A confirmed update is downloaded and verified against GitHub's asset digest,
  the published SHA-256 sidecar, repository and commit identities, the release
  manifest, runtime-file hashes, and all six portable schema hashes before it
  can be activated.
- Activation uses an atomic pointer update. Restart is limited to one handoff,
  the prior complete version remains available for rollback, and version
  removal is always explicit and protects current or unproven rollback state.
- The managed launcher and lifecycle core are exercised on Ubuntu, macOS, and
  Windows in GitHub Actions.

## Extraction and provenance

- The paired Bundle emits `coursecraft.run/1` receipts containing source
  fingerprints, producer identity, schema checksums, step outcomes, and output
  checksums. Combined Wizard packages preserve the exact Bundle version,
  repository, ref, and commit in that receipt even though distributed source
  trees do not contain `.git` metadata.
- Versioned, additive activity and structure contracts preserve unknown vendor
  kinds and future evidence instead of forcing unseen exports into a closed
  list of shapes.
- Welcome/getting-started syllabus links are recognized both as direct manifest
  items and as links nested in package-local HTML. The allow-listed fetch is
  supplemental, checksum-bound, and non-fatal; package-local course evidence
  remains authoritative.
- A regression gate protects learning-objective and learning-material headings
  whose visible text is wrapped in inline elements such as `strong` or `span`.

## Distribution choices

- `blueprint-wizard-managed-v2.8.0.zip` is recommended for new installations.
- `blueprint-wizard-v2.8.0.zip` remains available for existing portable users,
  offline import, and as the verified payload used by managed updates.
- Both ZIPs have independent `.sha256` sidecars and embedded manifests binding
  them to the exact source commits and Bundle contracts.

The v2.8.0 ZIPs are unsigned and are not notarized. macOS Gatekeeper, Windows
SmartScreen, or institutional device policy may therefore require an explicit
trust action. The release does not weaken those controls; broader distribution
remains gated on a future signing/notarization decision.
